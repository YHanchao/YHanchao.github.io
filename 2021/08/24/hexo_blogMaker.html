Hexo博客网站指北：如何建立属于自己的甜品站

 
  • 2021年 08月24日

写在前面：使用Hexo建立博客的教程在网上可以说是一搜一大把，可仔细一看基本都是互相抄。不过嘛本来这个东西就已经很简单了，所以这种同质化也能理解。这次这份甜品会写一些自己踩的坑以及相关的解决方案，希望能够大致说清楚如何建立这样一个网站。

搭建博客站点的整体思路

如何将自己的博客/文章/笔记公开发布呢？目前已经有不少成熟的解决方案，比如公众号/CSDN/知乎等等，但是在这些平台上总是有一种让自己处于被动位置的感觉，内容发布后的管辖权似乎就离自己越来越远了。于是就有了自己做一个网站来公开自己的内容的想法，这样整个网站归自己所有，自然也不会担心因为各种各样的原因而被下架。

但是对于写作的人而言，我只想写好一篇文章，然后这篇文章就能够漂漂亮亮地呈现在页面上，对于中间的实现步骤则希望越简单越好。Hexo恰好就是一个能够满足我大部分需求的解决方案，我只需要对其做一些配置，然后把文章的Markdown文本放到目录里面，之后生成、发布就好了。

那么主要的问题就集中在如何配置Hexo、文章发布到哪里（服务器）以及如何关联到自己的域名上这三者上了。下面就主要围绕这三个话题展开，同时附赠自己遇到的一些问题及其解决方案。

PS：本人只有Windows，所以下面都是基于Windows的。不过大多数流程都差不多，Mac党大概也可以参考（Linux党都有Linux了应该不会看我的博客吧hhh）

首先先附上Hexo的官方文档（要查阅肯定是官方文档最全面啦，有需要还可以去Github页的issue里翻一翻）

前期准备：Hexo的安装

那么先来安装这两个软件：

  • Git：绝大部分命令均需要通过Git Bash来实现，包括之后的发布。
  • Node.js：Hexo是依赖Node.js实现的，所以需要安装Node.js以及npm

安装的时候记得添加到环境变量（PATH）中，之后可以在CMD（命令提示符、Powershell等）里面通过查看版本来检查一下有没有安装成功

git --version
node -v
npm -v

如果安装好了的话就可以愉快地开始安装Hexo了，先右键打开Git Bash输入这条命令安装Hexo的核心组件（这个命令里面的$是原本Git Bash里面就有的，只不过是再强调一下这个是在Git Bash里面操作的而非CMD，之后也是！）

$ npm install -g hexo-cli

印象中这里由于已经设置了Node.js的环境变量，所以这里应当会安装到Node.js的目录下。

博客的初始化与自定义

博客的雏形

首先选定准备存放博客的目录，在这个目录下打开Git Bash输入如下命令：

$ hexo init <folder>
$ cd <folder>
$ npm install

第一条：初始化一个文件夹（当然这里的<folder>就是文件夹的名字，比如说可以是Blog）

第二条：命令行转到刚刚新建的文件夹下，这里的<folder>就是刚刚那个文件夹的名字

执行过上述步骤之后指定的文件夹下的目录将会是：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

其中_config.yml将会是下面自定义博客的过程中经常要用到的。source/_posts/文件夹下将会存放博客的markdown文件，其他几个文件夹的含义不妨直接查看文档，这里就不写了（

这样，博客的框架就搭建起来了。下面你可以在Git Bash里输入这两行命令查看现在你的博客的样子：

$ hexo g
$ hexo server

其中第一条是生成静态网页，第二条是建立一个本地服务器以预览你的博客，打开页面http://localhost:4000/就能看到你的博客了

那么下面就来看看如何让自己的博客变得更漂亮！

博客的初步自定义

注意：这里仅介绍初步使用Hexo的主题以及Hexo原生支持的一些功能。不同主题的作者事先实现的功能不同，使用语法也不相同，后续建议查阅对应主题的文档。

这里主要来看站点根目录下的_config.yml，这个文件是整个博客最“全局”的设置，博客中最主要的部分都是在这里完成的自定义。

不过在开始说之前，还想提另外两样东西。如果不出意外的话，此时同目录下还能找到config.landscape.yml，事实上如果后续安装了其他主题之后themes/<themeName>/下还会有一个_config.yml。hexo在生成静态网页时会同时使用上面提到的三个config，可以认为博客根目录下的_config.yml定义了整个博客的公共变量（如网站的名称等等），而主题下的_config,yml则是主题内的局部变量。一般来说有这两个config就够用了，前期不是很了解Hexo的情况下知道有这两个设置文档就足够了。

Hexo对这些设置文档的解释

我们强烈建议你将所有的主题配置集中在一处。如果你不得不在多处配置你的主题，那么这些信息对你将会非常有用：Hexo在合并主题配置时，Hexo配置文件中的theme_config的优先级最高，其次是_config.[theme].yml文件，最后是位于主题目录下的_config.yml文件。

这里在官方文档说的很清楚了，这里只需要简单说几个最初很可能用到的几个部分。

首先是这个文档的写作规范。文档默认通过两个空格的缩进来匹配，一般冒号:之后还需要跟随一个空格，比如在我的设置文本中有一部分是这样的：

# Site
title: 小闰的甜品站
subtitle: ''
description: 慢慢来，不要怕。点心总是会有的
keywords:
author: 小闰
language: zh-CN
timezone: ''

在这个文档里主要需要关注的就是Site部分、theme（设定主题）以及deploy（博客发布），具体内容文档内部写的比较详细了。

安装主题：给博客换装

这里是官网的主题页面

Hexo官网上有不少开发者制作的主题，选一个自己喜欢的点进去安装就好了。目前来看网上有不少Next主题的教程，更早以前甜品站也确实用的是这一款。这个主题对绝大多数你可能用的到的功能都已经做了适配，用起来算是比较无脑，美化起来也算比较方便。而且作为一款一直维护的主题，Next的相关资源也很丰富、bug也一直在修，总之用Next开始接触Hexo博客也算是一个不错的想法。

注意：不同主题各自实现的功能不完全相同，而且使用的语法也不完全相同，这也使得主题迁移需要一些时间成本。

那么下面就以安装Next为例介绍，其他主题同理。

如何使用Hexo写作文章

注意：你需要先事先了解Markdown的基本语法，不过这个语法非常简单，大约五分钟就可以基本掌握了

依然先附上官方文档

新建一个文档

基本操作就是使用命令

hexo new [layout] 文件名字

其中layout如果省略的话，就会默认生成到source/_posts文件夹下，最后生成的时候就会渲染这个文件。

开始写作：Front Matter

Front-matter是文件最上方以---分隔的区域，用于指定个别文件的变量，比如这篇文章包括：

---
title: Hexo博客网站指北：如何建立属于自己的甜品站
date: 2021-08-24 23:04:26
categories: 其他笔记
---

事实上，Hexo和主题已经事先定义了这些关键字，它们使用的语法和上面YAML一致（事实上这个就相当于对每个页面的_config）。你可以在文档中查询有哪些关键字及其功能与默认值。

之后你就可以愉快地开始写博客了！

使用Github Pages发布博客

发布博客的基本思路就是将public里面的文件上传到服务器上，之后访问服务器即可看到和之前在http://localhost:4000/上看到的内容。

首先是服务器选择。这里我找了不少云计算商家，在这里购买一个服务器的价格还是比较贵的。不过人家那些服务器都主要是服务于云计算、云存储等较为大型的场景，而这个博客只是一个静态页面，可以说没有计算需求，甚至各大厂商提供的对象存储服务（如华为云OBS、阿里云OSS）都可以实现静态页面访问。

所以我们只需要一个“轻量”的解决方案，最后试了一圈发现还是Github Pages最好用，理由如下：

  • Github Pages是一项免费的服务
  • Git可以方便地提交到Github上
  • Github建立在国外的服务器，域名无需备案即可访问（如果使用各大云计算厂家提供的服务器则需要先进行备案才能访问）

当然也有一些缺点，比如访问速度较慢（虽然说可以使用CDN加速，但是国内CDN太贵，国外的不敢用orz），不过有的部分还有办法解决，这个之后会写。

开始发布

首先安装发布到Github上的插件

$ npm install hexo-deployer-git --save

在博客根目录下的_config.yml中找到这一段

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy: 
  type: git
  repo: git@github.com:YHanchao/YHanchao.github.io.git
  branch: master

把里面repo和branch的内容填写为你创建的仓库的信息。现在Github创建的页面默认branch是main而不是master，这一点需要注意一下

保存好你的设置，运行命令

$ hexo d

成功发布之后你就可以访问username.github.io看到你的博客了。

之后你将会经常用到

$ hexo clean
$ hexo g
$ hexo server
$ hexo d
$ hexo new 这是一篇博客

第一个是清除public文件夹，第二个是生成静态博客，也就是生成public文件夹，第三个是在本地运行服务器预览当前页面，第四个是发布博客，也就是把public文件夹发布到github上，最后一个就是新建一篇博文。

GitHub Pages是Github提供的一个静态站点托管服务，它可以把上传到Github仓库上的静态文件（HTML、CSS、Javascript……）使用http协议构建为一个站点。每个用户可以创建唯一一个username.github.io作为自己的站点，不过在此之后你依然可以创建其他仓库并且开通Github Page，只不过这时你需要访问username.github.io/reponame才能看到你的网站。 这就是说在熟悉了操作之后你可以用个人站点搭建主站，之后使用项目站点搭建各种二级页面、子站点……

注意：按照Github的协议，项目和网站的大小不要超过 1GB，也不要过于频繁的更新网站的内容（每小时不超过 10 个版本），每个月的也要注意带宽使用上限为 100GB。

域名的选购与绑定

上一步成功部署到Github之后，每次都需要输入username.github.io才能访问，相对来说还是有一点麻烦，于是就有了买一个自己的域名绑定上去的想法。

国内现在基本上所有的云计算厂商都提供域名注册服务，你可以在华为云、阿里云、腾讯云这些厂商的域名注册服务里选购一个价格便宜的域名（亲测百度云不提供这项服务）。域名选购的时候一般是需要进行实名验证、邮箱验证、各种验证，不过这个流程基本上就和注册账号差不多，按照提示要求一步步做就行了。

我之前是用华为云买的域名，不过其他云的操作方式都差不多（最近把域名搬到阿里云了）。下面就拿华为云为例说明怎么绑定域名。

一般来说绑定方法有两种，一种是设置CNAME，另外一种是使用ip。我之前使用的是CNAME方式，不过下面还是把两种都罗列一下（

  • 打开域名解析

  • 添加一条记录：上面两种都可以选

  • 先来看使用添加记录集

主机记录填的是之后你希望访问的域名，向上面这样什么都不填的话就是访问yhanchao.cn；类型选择CNAME，就是把域名指向另一个域名；值就填你的username.github.io。最后点确定即可。

  • 再来看使用快速添加解析

选择网站解析，之后选择域名，填入网址即可。

IP方式和CNAME大同小异，就简单提一下就好了。

  • 获取IP地址：在命令提示符（CMD、Powershell）输入ping username.github.io，输出里面包含的那一串数字就是IP地址

  • 使用添加记录集：类型选择A，值填入IP地址

在上面这些做完之后，在博客的source文件夹下添加一个文件CNAME（没有后缀名！），在里面填入刚刚绑定的域名（如：yhanchao.cn），将这个文件上传到Github仓库的根目录下。这样做是为了保证之后上传博客内容的时候CNAME文件不会被删除。

最后可以检查一下仓库设置。打开仓库的Settings > Pages，检查一下有没有绑定上去，没有的话填一下Custom Domain，需要https的话也可以开启。

好了，现在你已经可以通过你绑定的域名来访问你的博客了！开始发布吧！

我遇到的问题与解决方案

服务器选择

直接上结论：

  • 国内服务器：价格贵，博客没有算力需求，浪费资源；域名指向国内服务器需要备案。
  • 使用云计算平台的对象存储服务发布：访问速度远快于Github，但是也会遇到备案问题。大多云服务提供商不会为指向对象存储服务的域名提供备案服务
  • Github Pages + 对象存储服务：Github Pages作为发布静态博客的服务器可以免费使用，域名无需备案；图像、视频等各类资源可以放到对象存储服务上快速访问（如果只有图像的话可以考虑找一个靠谱的图床，如果有下载站搭建需求的就使用对象存储服务好了）

最后我选择了第三个方案，虽然造成了一定的运行成本，但是这一成本属于可控范围之内，不会带来过大的经济负担，便宜大碗。

子站点建立

我建立了其他的子博客、下载站，希望通过Github Pages发布。解决方案如下：

首先在Github里分别建立存放这些静态网站的仓库，开启Github Pages，和前文一样将静态网站的内容发布到Github上。

之后在域名解析中添加一条记录，比如我希望子站点的域名是foobar.yhanchao.cn，使用CNAME方式指向yhanchao.github.io，之后在GitHub的项目仓库中添加CNAME文件，内容为foobar.yhanchao.cn，这样即可访问。

上述操作之后只能通过http方式访问，如果使用Chrome浏览器的话会显示站点不安全，于是需要开启https访问。GitHub Pages仅为主站点提供了https访问的证书，所以需要自行为其他站点安装SSL证书。在云计算平台购买SSL证书的时候只需要购买免费版即可，按照操作进行验证之后几分钟之内就能审批下来。

之后就可以使用https进行访问了。
